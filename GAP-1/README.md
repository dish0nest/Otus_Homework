## Компоненты стенда
```
В стенд входят:
  1. 2 Виртуальные машины(ОС Debian 12)
    - Execute
    - Monitoring
  2. В VM Execute входят:
    - Nginx (вне Docker контура)
    - Node-exporter (вне Docker контура)
    - Docker контур:
      - Nginx-exporter
      - cAdvisor
      - blackbox-exporter
      - 2 контейнера с приложением
      - СУБД postgresql
      - psql-exporter
  3. В VM Monitoring входит:
    - Prometheus (Docker)
```

## Шаги выполнения
```
1. Установка Nginx
  - Установка происходит с помощью команды sudo apt install nginx
  - Конфиг Nginx представлен в файле nginx.conf
  - * Для выполнения задания с добавлением SSL, было решено весь трафик перенаправлять через nginx
  - В конфиге nginx был создан upstream для балансировки запросов к приложению
    Был настроен сертификат, секретный ключ, учетная запись (указан файл .htpasswd) + настроены протоколы и шифрование
    Было настроено перенаправление трафика от Nginx к контейнеру (в docker-compose.yml порты были открыты только на localhost, извне доступа к ним нет)
  - Был открыт stub_status по порту 8080 для сборка метрик
```
```shell
2. Создание сертификата и ключей
  - Для создания сертификата и ключей использовалась команда:
      openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout server.key -out server.crt
  - Сертификат был передан на VM Monitoring для дальнейшего использования и обеспечения аутентификации
```
```
3. Сбор метрик
  - Для сбора метрик с VM используется node-exporter (https://github.com/prometheus/node_exporter).
      Находится вне Docker контура, исполняемый файл был загружен с репозитория и обернут, как сервис (node-exporter.service).
      * Так же для настройки безопасности используется web-config.yml, который передается флагом --web.config.file
  - Для проверки работоспособности контейнеров с приложением используется blackbox_exporter (https://github.com/prometheus/blackbox_exporter)
      Находится в Docker контуре, поэтому был взят образ
      Настраивается с помощью blackbox.yml, в котором прописано
        По какому протоколу проводить проверку
        С какой частотой
        Какие ответы от сервера считать валидными и какой метод применять
  - Для мониторинга СУБД используется образ postgres_exporter (https://github.com/prometheus-community/postgres_exporter)
      Для подключения к СУБД в конфиге docker-compose.yml прописываются переменные, которые указывают:
        Строку подключения
        Логин и пароль
  - Для сбора метрик с Nginx был использован образ nginx-exporter (https://github.com/nginx/nginx-prometheus-exporter)
      В docker-compose прописывается переменная URI для сбора метрик
  - Для мониторинга Docker контура был использован образ cAdvisor (https://github.com/google/cadvisor)
      В docker-compose в volumes прописываются директории для сбора метрик по используемым ресурсам
```
```
4. Сбор docker-compose
  - Для поднятия стенда был написан docker-compose, основываясь на п.3 + были настреоны зависимости сервисов друг от друга
```
```
5. Prometheus
  - Prometheus разворачивался в контейнере
  - Для Prometheus был создан prometheus.yml, в котором прописано:
      Время сбора метрик в блоке global
      Для сбора метрик было определено 2 блока job_name
        - node - блок, который собирает все метрики по endpoint metrics (все таргеты и label для них прописаны в файле target.json)
            Для обеспечения безопасности выбирается схема https
            Указывется путь к сертификату в tls_config
            Указываются данные от Ученой записи в basic_auth
        - blackbox_health - блок, который собирает данные о том, "живы" контейнеры с приложением или нет, по пути /probe
            Указывается схема https, путь до сертификата и basic_auth
            Так же в params указывается модуль, который был указан в blackbox.yml
            В targets указываются IP адрес + порт контейнеров, которые нужно опросить
            В relable_configs указываются данные для сохранения labels
```
